# 🗺️ AI Desktop 路线图

> 从"聊天窗口"到"Linux 桌面管家"

## 现状（2026-06）

- ✅ Tauri v2 + React + Vite 脚手架已完成
- ✅ 侧边栏 UI 骨架 + 设置弹窗 + 欢迎向导
- ❌ 没有真正的 API 调用（目前是 mock）
- ❌ 没有系统操作能力
- ❌ 产品方向模糊（"聊天侧边栏"定位不清）

## Phase 1：跑通链路 — "能对话" 🎯 当前

**目标：让 AI Desktop 真正用上本地模型，能聊天、能执行基础命令。**

| 任务 | 内容 | 优先级 |
|------|------|:------:|
| 1.1 接真实 API | 把 `handleSend` 的 mock 换成 `fetch` 调本地 LLM | 🔴 |
| 1.2 预设 provider | 默认 `http://localhost:8080/v1`，开箱即用 | 🔴 |
| 1.3 流式输出 | Server-Sent Events 流式显示回复（打字机效果） | 🟡 |
| 1.4 系统命令 | Rust 端暴露 `run_command`，让 AI 能执行 shell | 🟡 |
| 1.5 错误处理 | 模型挂了、网络不通、命令失败 → 友好的错误提示 | 🟢 |

**验收标准：** 打开软件 → 输入"ls ~"→ AI 理解并执行 → 显示目录列表

## Phase 2：三种模式 — "能帮人" 🎯 下一个

**目标：从纯聊天进化到模式化交互，不同用户用不同方式。**

### 2.1 📖 教我模式

AI 分步解释操作，每一步：
- 解释"这一步在做什么"
- 显示将要执行的命令
- 用户点"确认"才执行或"跳过"看下一步

适用于：首次装软件、配系统、学命令的新用户

### 2.2 ⚡ 帮我模式

AI 全自动执行：
- 用户描述需求 → AI 拆解步骤
- 批量执行 → 实时反馈进度
- 最后给出总结："已安装 Firefox（版本 130），快捷方式已放在桌面"

适用于：想省事的所有用户

### 2.3 👀 监控模式

后台常驻系统托盘：
- 监听终端命令退出码（非 0 → 弹窗解释）
- 监听磁盘/内存阈值（空间不足 → 建议清理）
- 截屏错误代码 → 自动识别 → 给修复方案

适用于：所有人（包括你）

| 任务 | 内容 | 优先级 |
|------|------|:------:|
| 2.1 模式切换 UI | 顶部三个模式标签，切换后 system prompt 不同 | 🔴 |
| 2.2 教我模式 | 分步确认组件 + 命令执行回滚 | 🔴 |
| 2.3 帮我模式 | 批量执行 + 进度提示 + 结果汇总 | 🟡 |
| 2.4 后台托盘 | 系统托盘常驻 + 快捷键呼出 | 🟡 |
| 2.5 异常捕获 | 监听 stderr / 磁盘 / 进程退出码 | 🟢 |

**验收标准：** 新手能用"教我模式"装好一个软件；老手能用"帮我模式"一键配环境

## Phase 3：打包分发 — "能推广"

**目标：让 Linux 新手也能装上用。**

| 任务 | 内容 | 优先级 |
|------|------|:------:|
| 3.1 AppImage 构建 | `cargo tauri build` 输出 AppImage | 🔴 |
| 3.2 .deb / .rpm | 主流发行版包格式 | 🟡 |
| 3.3 GitHub CI | Release 时自动构建所有格式 | 🟡 |
| 3.4 文档完善 | 使用教程、FAQ、常见问题排查 | 🟢 |
| 3.5 配套脚本 | 一键安装 llama.cpp + 下载模型 | 🟢 |

**验收标准：** 一个刚装好 Ubuntu 的人，看完 README 能在 10 分钟内用上

---

## Phase 4：社区化 — "能长大"

**目标：有人用、有人贡献、有人提需求。**

| 任务 | 内容 | 优先级 |
|------|------|:------:|
| 4.1 CONTRIBUTING.md | 贡献指南、代码规范、PR 流程 | 🟡 |
| 4.2 插件系统 | 社区可写自定义"技能"（类似 OpenClaw plugin-skills） | 🟢 |
| 4.3 多模型支持 | 除 llama.cpp 外支持 Ollama、vLLM 等 | 🟢 |
| 4.4 国际化 | 英文 README + 英文界面 | 🟢 |
