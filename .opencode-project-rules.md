# AI Desktop - OpenCode 开发规则

## Rust / Tauri v2 关键规范

### 1. 宏调用
- 正确: `tauri::generate_handler!(run_command, ...)`
- 错误: `tauri::generate_invoke_handler!(...)` ← 不存在

### 2. Command 参数传递
- 正确: `.arg(command)` `.arg("/tmp/screenshot.png")`
- 错误: `&command` `&"/tmp/screenshot.png"` ← 语法错误

### 3. 系统托盘（Tauri v2）
- 用 `TrayIconBuilder::new()`（无参数）
- 构建: `.build(app)?` 返回 `Result`
- 事件: `TrayIconEvent::Click { button: MouseButton::Left, .. }`
- 获取窗口: `tray.app_handle().get_webview_window("ai-sidebar")`
- 需要在 `Cargo.toml` 的 `tauri` 依赖加 `"tray-icon"` feature

### 4. tauri.conf.json
- `systemTray` 是**顶层键**，不在 `app` 内部
- 但 Tauri v2 推荐用 Rust API 配置托盘，json 中可不设

### 5. frontendDist 路径
- 相对 `src-tauri/` 目录: `"../dist"`（不是 `"../../dist"`）

### 6. 每次改完 Rust 后
- 必须跑 `cargo check` 验证编译
- 有错立即修复，循环直到通过

## 前端规范

### 1. API 调用
- 默认: `http://127.0.0.1:8082/v1/chat/completions`
- 模型: `coder-next`
- 使用 SSE 流式 `stream: true`
- system prompt 必须包含在 messages 数组第一项

### 2. 欢迎向导
- 推荐 provider: llama.cpp（不是 Ollama）
- 默认 URL: `http://127.0.0.1:8082/v1/chat/completions`
- API Key: 空（本地模型不需要）

### 3. 样式
- 深色主题用 CSS 变量
- 保持 sidebar 布局不变

## 构建

- 打包: 先跑 `npm run build` 再 `cargo tauri build`
- 产物在 `src-tauri/target/release/bundle/`
