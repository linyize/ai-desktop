# AI Voice Terminal Gateway — OpenCode 开发规则

## 项目结构
- `voice-terminal/` 为项目根目录
- `gateway.py` — 主入口（WebSocket + HTTP 服务）
- `asr_engine.py` — ASR 封装（调用系统 whisper）
- `tts_engine.py` — TTS 封装（edge-tts / MOSS-TTS）
- `router.py` — 关键词路由选择 LLM
- `llm_client.py` — 统一 LLM 调用接口
- `audio_utils.py` — 音频工具函数
- `config.py` — YAML 配置加载
- `win_client.py` — Windows 客户端（录音+播放+WebSocket）
- `config.yaml` — 配置文件
- `requirements-gateway.txt` — Linux 端依赖
- `requirements-win.txt` — Windows 端依赖

## 技术栈
- Python 3.11+
- WebSocket: `websockets` 库（服务端异步）
- ASR: 调用 `whisper` CLI 或 `openai-whisper` Python 库
- LLM: OpenAI 兼容 API（本地 coder-next / 35b 或云端 API）
- TTS: `edge-tts`（默认快速）/ MOSS-TTS（声音克隆）
- 配置: YAML（`pyyaml`）

## 通信协议
- WebSocket JSON 消息（详见 IMPLEMENTATION_PLAN.md）
- REST API HTTP

## 构建和测试
- 每个组件独立测试：`python3 test_<component>.py`
- 不依赖复杂框架，保持单文件架构
- ASR 调用 `whisper` CLI（系统路径），TTS 用 `edge-tts` CLI 或 Python SDK

## 命名规范
- 类名: PascalCase, 函数/变量: snake_case
- 异步函数用 async/await
- 日志用 print() 或 logging 模块
